# ROADMAP.md

> 개인 홈페이지 제작 기획서 — Claude Code가 본 문서를 기반으로 작업합니다.
> 본문 작업 전 반드시 `CLAUDE.md`의 Plan Mode 동작 지침을 따를 것.

---

## 1. 프로젝트 개요

### 1.1 한 줄 정의
**"PM 직무 어필을 메인 동선으로 하면서, 한 사람의 다층적 정체성을 함께 보여주는 1인 포트폴리오 사이트."**

### 1.2 목적
- 1차: PM(프로젝트 매니저) 직무 지원 시 포트폴리오로 사용
- 2차: 채용 외 일반 네트워킹·자기소개 자산으로 제너럴하게 사용
- 3차: 향후 코드랩아카데미 교육 산출물 및 이후 프로젝트를 **누적**시켜 나갈 거점

### 1.3 대상 독자 (우선순위)
1. **PM 채용담당자/리쿠르터** — 메인 동선 (Home → Projects → Contact)
2. **협업 제안자/네트워킹 상대** — Home → About → Experience
3. **호기심 있는 일반 방문자** — About → Side Quests로 진입

### 1.4 핵심 메시지 (positioning statement)
> "다양한 도메인에서 문제 정의·기획·실행을 경험한 제너럴리스트.
> 국제 협업, 공공 R&D, 지역 사회 문제, 본인 사업 운영을 거치며
> '현실 문제를 정의하고 해결하는' 한 가지 동작을 다양한 맥락에서 반복해 왔습니다."

### 2.2 페이지별 콘텐츠 구성

## 4. 기술 스택

### 4.1 프레임워크
- **Astro** (정적 사이트 생성, Content Collections 활용)
- **Tailwind CSS** (디자인 시스템)
- **TypeScript** (Astro 기본)

### 4.2 콘텐츠 관리
- **Content Collections**: `src/content/projects/`, `src/content/side-quests/`, `src/content/experience/` 디렉터리에 `.mdx` 파일로 관리
- 각 컬렉션마다 schema 정의해서 일관된 메타데이터 강제 (제목, 날짜, 태그, 요약 등)
- 새 항목 추가 = `.mdx` 파일 추가 → 빌드 시 카드/상세 페이지 자동 생성

### 4.3 인터랙션
- **Framer Motion** 또는 CSS 기반 절제된 스크롤·호버 모션
- 과한 인터랙션 금지 (가벼워 보일 위험)

### 4.4 배포
- **GitHub Pages**
- 저장소명: `lynstory.github.io` (사용자 도메인)
- 배포 URL: `https://lynstory.github.io`
- GitHub Actions로 자동 빌드·배포 (Astro 공식 워크플로우 사용)

> [가정] 본인 도메인은 추후 구매 시 CNAME 파일 추가 한 줄로 연결.

### 5.2 타이포그래피
- 영문: Inter / IBM Plex Sans 계열

- 큰 헤딩에 적당한 무게감, 본문은 읽기 편한 크기

- 충분한 여백
- 이미지보다 타이포·구조 중심

### 5.4 다크모드
- 1차 출시 범위: **라이트모드 우선**
- 다크모드는 V2 옵션 (필수 아님)

### 5.5 프로필 사진
- Hero(우측)와 About 페이지 상단에 동일/유사 이미지 사용
- 정사각형 또는 원형 크롭
- 흑백 또는 컬러 처리는 시안 단계에서 결정
- 파일 위치: `/public/profile.jpg` (또는 `.webp`)

## 6. 개발 단계 / 마일스톤

### Phase 1 — 골격 (Skeleton)
- Astro 프로젝트 초기화, Tailwind 연결
- 기본 레이아웃 + 네비게이션
- 5개 페이지 라우팅: `/`, `/about`, `/experience`, `/projects`, `/contact`
- Side Quests 라우트 (`/side-quests`) 별도 추가
- 빈 페이지로라도 모두 클릭 가능 상태

### Phase 2 — Content Collections 구조 구축
- `src/content/` 컬렉션 schema 정의
- Projects 카드 템플릿 (Astro 컴포넌트)
- Side Quests 카드 템플릿
- Experience 타임라인 컴포넌트
- 더미 데이터 1~2개로 동작 확인

### Phase 3 — 콘텐츠 입력
- About 본문 작성 (간접·은유 톤 유지)
- Experience 항목 입력 (검증된 12개 항목)
- Projects 초기 3개 케이스 스터디 작성
- Side Quests 3개 항목 작성

### Phase 4 — 디자인 마감
- Hero 비주얼 (프로필 사진 포함)
- 타이포·여백 튜닝
- 인터랙션·애니메이션 절제 적용
- 모바일 반응형 검수

### Phase 5 — 배포
- GitHub Actions 워크플로우 설정
- `lynstory.github.io` 저장소에 push
- 도메인·SEO 메타데이터 점검

### Phase 6 — 누적 운영 (지속)
- 코드랩 산출물 생기는 대로 `.mdx` 추가
- 이후 프로젝트도 같은 템플릿으로 추가
- About / Experience는 분기 단위 점검

## 7. 누적 운영 정책

이 사이트는 **계속 자라나는 사이트**입니다. 다음 정책을 따릅니다.

### 7.1 새 프로젝트 추가 워크플로우
1. `src/content/projects/[slug].mdx` 파일 생성
2. 정해진 schema의 frontmatter 작성 (제목, 날짜, 태그, 요약 등)
3. 본문은 Projects 카드 템플릿(§2.2 Projects) 따라 작성
4. 빌드 → 자동으로 카드와 상세 페이지 생성

### 7.2 정렬 정책
- **상단 고정 3개**: frontmatter에 `featured: true` 플래그
- 그 아래는 **최신순**(`date` 기준 내림차순)
- 코드랩 산출물이 누적되어도 강한 케이스가 묻히지 않도록

### 7.3 카테고리·태그 정책
- 카테고리는 §2.2 Projects에 정의된 5개 고정 (확장 시 ROADMAP 업데이트)
- 태그는 자유

## 10. 성공 기준

- ✅ PM 채용담당자가 5분 내에 핵심 역량과 경험을 파악 가능
- ✅ 새 프로젝트 추가가 `.mdx` 파일 1개 작성으로 끝남
- ✅ 모바일에서 깨짐 없이 작동
- ✅ 첫 인상이 "밋밋하지 않고", 그러나 "가벼워 보이지도 않음"
- ✅ Side Quests에 진입한 방문자가 "이 사람 입체적이네"를 느낌
